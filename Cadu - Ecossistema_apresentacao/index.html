Forma mais simples (recomendada)

Se não quiser usar o terminal à mão:

  1. No Finder, abra a pasta deste projeto no Ambiente de Trabalho (ou onde a guardou).
  2. Procure o ficheiro abrir-instituto.command.
  3. Primeira vez no Mac: clique com o botão direito no ficheiro → Abrir → confirme “Abrir”. (Isto evita o bloqueio de segurança.)
  4. Depois pode usar duplo clique. Abre-se uma janela preta (Terminal), o site prepara-se e o browser abre sozinho.
  5. Não feche a janela preta enquanto estiver a ver o site. Para parar: na janela preta, prima Ctrl+C.

Precisa de ter o Node.js instalado (grátis): nodejs.org → botão LTS.

Abrir o projeto no navegador (manual)

As rotas Next.js (/, /plano, /memory…) só funcionam com o servidor a correr. Abrir pastas no Finder não inicia o site.

1) Terminal (na pasta deste projeto)

Primeira vez (instalar dependências):

npm run install-web

Subir o site:

npm run dev

Ou, se preferir entrar na pasta web:

cd web
npm install
npm run dev

2) No navegador

  • http://localhost:3000 (atlas)
  • http://localhost:3000/plano (apresentação Master Plan, editável em React)

3) Só HTML (sem servidor)

ecossistema_cdu_presentation.html: versão estática (comentário no ficheiro aponta para o código no Next).

Se localhost:3000 não abrir: confirma que o terminal mostrou “Ready” e que nenhuma firewall bloqueia a porta. No Cursor, usa Terminal → New Terminal e corre os comandos acima.
